# Shadow Copy-Trader Platform - Project Plan

## 1. Project Overview

**Shadow Copy-Trader** is a full-stack copy-trading platform for Polymarket that enables users to automatically copy trades from top-performing traders. The platform provides comprehensive analytics, performance tracking, and automated trade execution.

### Core Value Proposition
- **For Beginners**: Copy successful traders without needing market analysis skills
- **For Busy Users**: Automated trading while you focus on other things
- **For Learners**: Learn by observing top trader strategies

---

## 2. Tech Stack

### Frontend
- **Framework**: Next.js 14+ (App Router) with TypeScript
- **Styling**: Tailwind CSS + shadcn/ui components
- **Charts**: Recharts for data visualization
- **Icons**: Lucide React
- **Animations**: Framer Motion
- **State Management**: React Query (TanStack Query) + Zustand
- **Forms**: React Hook Form + Zod validation
- **Web3**: wagmi + viem for wallet interactions

### Backend
- **Framework**: Python FastAPI
- **Database**: PostgreSQL with SQLAlchemy ORM
- **Cache**: Redis for real-time data + Pub/Sub for WebSocket scaling
- **Task Queue**: Celery for background job processing
- **WebSocket**: FastAPI WebSocket with Redis Pub/Sub for horizontal scaling

### External APIs
- **Polymarket CLOB API**: Order placement and market data
- **Polymarket Gamma API**: Market discovery and analytics
- **Builder Relayer**: Gasless transaction execution via Safe proxy wallets

### Testing & DevOps
- **Backend Testing**: Pytest + pytest-asyncio
- **Frontend Testing**: Jest + React Testing Library + Cypress (E2E)
- **CI/CD**: GitHub Actions
- **Monitoring**: Prometheus + Grafana
- **Logging**: Structured logging with ELK stack / Sentry

### 3.6 Copy Ratio Explanation

**Copy Ratio** determines how your allocated capital mirrors the lead trader's position sizing relative to their portfolio.

**Example:**
- Your allocation to CryptoWhale: $1,000
- Copy Ratio: 50%
- CryptoWhale opens a position using 10% of their portfolio

**Calculation:**
- Your position size = $1,000 Ã— 10% Ã— 50% = $50

This approach ensures:
1. Position sizes are proportional to trader's risk appetite
2. Users can scale down risk by lowering the ratio
3. Large traders don't cause oversized positions for copiers

## 5. API Endpoints (Optimized)

### 5.1 Authentication (SIWE - EIP-4361)
```
GET    /api/auth/nonce              # Get challenge nonce
POST   /api/auth/verify             # Verify SIWE signature + issue JWT
GET    /api/auth/me                 # Get current user (requires JWT)
POST   /api/auth/logout             # Invalidate session
```

**Authentication Flow:**
1. Frontend calls `/api/auth/nonce` â†’ Backend returns unique nonce
2. User signs SIWE message with wallet (EIP-4361 format)
3. Frontend sends signature to `/api/auth/verify`
4. Backend verifies signature, creates/updates user, returns JWT
5. Frontend includes JWT in all subsequent requests

### 5.2 Traders (Consolidated)
```
GET    /api/traders                 # List traders with filters
       ?timeframe=30d
       &min_win_rate=60
       &sort=roi
       &page=1&limit=20

GET    /api/traders/top             # Get top 10 performers

GET    /api/traders/:address        # Get full trader profile
       ?include=positions,history   # Optional includes

       Response includes:
       - Basic info (address, first_trade_date)
       - Performance metrics (roi, win_rate, trades, etc.)
       - Risk metrics (max_drawdown, sharpe, profit_factor)
       - Category breakdown
```

## 7. Security Architecture

### 7.1 Authentication & Authorization

#### SIWE (Sign-In with Ethereum) Flow - EIP-4361
```
1. GET /api/auth/nonce
   Response: { nonce: "abc123..." }

2. User signs message in wallet:
   "shadow-copy-trader.com wants you to sign in with your Ethereum account:
    0x1234...5678

    Sign in to Shadow Copy-Trader

    URI: https://shadow-copy-trader.com
    Version: 1
    Chain ID: 137
    Nonce: abc123...
    Issued At: 2026-01-12T10:00:00Z"

3. POST /api/auth/verify
   Body: { message, signature }
   Response: { token: "jwt...", user: {...} }
```

#### JWT Authorization
- All protected endpoints require `Authorization: Bearer <jwt>` header
- JWT payload contains: `{ user_id, wallet_address, exp }`
- JWT expiry: 24 hours
- Refresh via re-authentication (SIWE)

#### Resource Authorization Policies
```python
# All endpoints enforce user-scoped access:
- GET /api/portfolio â†’ user.id == portfolio.user_id
- GET /api/copies â†’ user.id == copy_config.user_id
- PUT /api/copies/:id â†’ user.id == copy_config.user_id
- etc.
```

#### Safe Proxy Wallet Model
Each user gets a deterministic Gnosis Safe proxy wallet:
- **Ownership**: User's EOA is the sole owner
- **Platform Role**: Shadow platform is an approved "module" with limited permissions
- **Permissions**: Module can only execute trades within user-defined limits
- **Revocation**: User can remove module permission at any time

#### Execution Flow Detail
```python
# copy_engine.py - Core copy execution logic
async def execute_copy_trade(copy_config: CopyConfig, lead_trade: Trade):
    # 1. Validate copy configuration is active
    if not copy_config.is_active:
        return

    # 2. Calculate position size using copy ratio
    lead_portfolio_percentage = lead_trade.size / lead_trader.portfolio_value
    user_position_size = (
        copy_config.allocation *
        lead_portfolio_percentage *
        copy_config.copy_ratio / 100
    )

    # 3. Apply constraints
    user_position_size = min(
        user_position_size,
        copy_config.max_position_size,
        copy_config.remaining_allocation
    )

    # 4. Validate minimum trade size
    if user_position_size < MIN_TRADE_SIZE:
        return  # Skip if too small

    # 5. Execute via user's Safe wallet
    tx_hash = await trade_executor.execute(
        user_safe_address=copy_config.user.safe_address,
        market_id=lead_trade.market_id,
        side=lead_trade.side,
        size=user_position_size
    )

    # 6. Record position
    await create_copied_position(copy_config, lead_trade, user_position_size, tx_hash)

    # 7. Notify user
    await notify_user(copy_config.user_id, "trade_copied", {...})
```

#### Key Management
- **User Keys**: Never stored; user signs with their own wallet
- **Platform Keys**: Used only for HMAC signing to Polymarket API
- **Storage**: AWS Secrets Manager / HashiCorp Vault for production
- **Rotation**: API keys rotated every 90 days

### 7.3 API Security

- **Input Validation**: Pydantic schemas with strict types
- **CORS**: Restricted to frontend domain only

### 8.2 Copy Execution Algorithm

```python
async def process_new_trade(trader_address: str, trade: Trade):
    """Process a new trade from a lead trader."""

    # Get all users copying this trader
    copy_configs = await get_active_copies_for_trader(trader_address)

    for config in copy_configs:
        # Skip if auto-copy disabled
        if not config.auto_copy_new:
            continue

        # Calculate user's position size
        trader_profile = await get_trader_profile(trader_address)
        trade_percentage = trade.size / trader_profile.total_volume

        user_size = (
            config.allocation *
            trade_percentage *
            (config.copy_ratio / 100)
        )

        # Apply limits
        user_size = min(user_size, config.max_position_size)
        user_size = min(user_size, config.remaining_allocation)

        if user_size < MIN_TRADE_SIZE:
            continue

        # Execute trade
        try:
            position = await execute_trade(
                user_id=config.user_id,
                market_id=trade.market_id,
                side=trade.side,
                size=user_size,
                copy_config_id=config.id
            )

            # Update remaining allocation
            config.remaining_allocation -= user_size
            await config.save()

            # Set stop loss if configured
            if config.stop_loss_percentage:
                position.stop_loss_price = calculate_stop_loss(
                    entry_price=position.entry_price,
                    side=position.side,
                    percentage=config.stop_loss_percentage
                )

            # Notify user
            await create_notification(
                user_id=config.user_id,
                type="trade_copied",
                title="Trade Copied",
                message=f"Copied {trade.side} position on {trade.market_name}",
                data={"position_id": position.id}
            )

            # Broadcast via WebSocket
            await broadcast_to_user(config.user_id, "trade_copied", position)

        except InsufficientFundsError:
            await create_notification(
                user_id=config.user_id,
                type="error",
                title="Copy Failed",
                message="Insufficient funds to copy trade"
            )
```

## 11. Monitoring & Logging

### 11.1 Metrics (Prometheus + Grafana)

```python
# app/metrics.py
from prometheus_client import Counter, Histogram, Gauge

# Business metrics
trades_copied_total = Counter(
    'trades_copied_total',
    'Total number of trades copied',
    ['trader_address', 'status']
)

copy_execution_duration = Histogram(
    'copy_execution_duration_seconds',
    'Time to execute a copy trade'
)

active_copy_configs = Gauge(
    'active_copy_configs',
    'Number of active copy configurations'
)

# System metrics
api_request_duration = Histogram(
    'api_request_duration_seconds',
    'API request duration',
    ['method', 'endpoint', 'status']
)
```

### 11.2 Logging (Structured)

```python
# app/logging.py
import structlog

logger = structlog.get_logger()

# Usage
logger.info(
    "trade_copied",
    user_id=user_id,
    trader_address=trader_address,
    market_id=market_id,
    size=size,
```
